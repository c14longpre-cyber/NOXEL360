# NOXEL LIBRARY

A beginner-friendly internal knowledge base for NOXEL projects.

## Structure
- `entries/` contains one concept per file.
- `TEMPLATE.md` is the standard format for new entries.

## Naming convention
- Use kebab-case filenames:
  - `html-headings.md`
  - `npm-vs-yarn.md`
  - `what-is-an-api.md`

## Writing rules (simple)
- Keep it beginner-friendly.
- Use analogies when helpful.
- Keep it factual and practical.
- Avoid long walls of text.
